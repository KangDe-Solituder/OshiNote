OshiNote は、推し活の記録を整理するためのローカルファーストなデスクトップアプリです。

静かであたたかく、長く残せる自分だけの場所を作ることを目指しています。推しごとのプロフィールを作り、配信後の気持ちを書き残し、大切な画像資料を整理し、ノート・散ページ・手帳を少しずつ自分だけの思い出アーカイブにできます。

> Record memories, emotions, and stream impressions for your oshi.

## 現在の状態

OshiNote は現在、初期段階ながら基本的に利用できる状態です。デスクトップ基盤、ローカル SQLite データベース、推しスペース、リッチテキストノート、画像資料、トップレベルの手帳モジュール、三言語 UI が実装されています。

今後は主に次の三方向で開発を進めます。

- フレームワークの安定性、アニメーションの滑らかさ、リソース読み込み、メモリ使用量、ビルドチェックの改善。
- Note、Illustration、Journal、Book、Postcard などの作成・整理体験の改善。
- テンプレート/素材管理、スタンプシステム、より自由な手帳作成など、推し活らしい機能の追加。

## 機能概要

### 推しスペース

- アバター、説明、テーマカラー、関連リンクを含む推しプロフィールを作成・管理できます。
- 各推しスペースでノート、画像資料、タグを確認できます。
- ホーム画面では最近のノートと最近の画像資料を一覧できます。

### ノート

- TipTap ベースのリッチテキストエディタ。
- 太字、斜体、下線、取り消し線、文字サイズ、フォント、文字色、ハイライト、絵文字に対応。
- 日付、カテゴリ、関連する推し、ソースリンクなどのメタ情報に対応。
- お気に入り、削除、保存、アーカイブ状態、タグ、キーワード検索に対応。

### 画像資料

- ローカルの PNG、JPEG、WebP 画像を取り込めます。
- 取り込んだ画像はアプリのローカルデータディレクトリにコピーされ、データベースで管理されます。
- 全体の画像資料ページ、または特定の推しスペース内で閲覧できます。
- カード、密集グリッド、リストの複数レイアウトに対応。
- カードをクリックすると右側の詳細パネルが開き、メタ情報、編集、削除などを操作できます。
- 詳細画像をクリックすると全画面プレビューを開けます。
- 画像資料を削除すると、アプリがコピーしたローカルメディアファイルも削除されます。

### 手帳

- 手帳は特定の推しスペースに固定されず、トップレベルのモジュールとして扱われます。
- 手帳管理ページでは、本と散ページを書架のようにまとめて管理できます。
- サイドバーから手帳作成画面を直接開けます。
- ノートや画像資料をキャンバス上に配置できます。
- 右側ドロワーでページタイトル、日付、説明、背景、収納先などを設定できます。
- ページは Book に保存することも、loose page として残すこともできます。
- 古い推しスペース内の手帳ルートは、新しいトップレベル手帳モジュールへ互換的にリダイレクトされます。

### 国際化

- English、简体中文、日本語に対応。
- デフォルト言語は English です。
- 設定で選択した言語はローカルに保存され、次回起動時に復元されます。

### 外観とモーション

- 複数のビジュアルテーマに対応。
- UI の文字サイズとアニメーション速度を調整できます。
- ドロップダウン、ドロワー、ダイアログ、詳細パネル、全画面プレビューは統一された overlay 階層とモーション規則を使います。

## インストールと実行

### 必要環境

事前にインストールしてください。

- Node.js と pnpm
- Rust
- Tauri が必要とする各 OS のシステム依存関係

詳細は [Tauri Prerequisites](https://v2.tauri.app/start/prerequisites/) を参照してください。

Windows では Microsoft C++ Build Tools と WebView2 Runtime が必要になる場合があります。最近の Windows には WebView2 が含まれていることが多いですが、Tauri の公式ドキュメントを基準にしてください。

### ソースコード取得

```bash
git clone https://github.com/KangDe-Solituder/OshiNote.git
cd OshiNote
pnpm install
```

SSH でも取得できます。

```bash
git clone git@github.com:KangDe-Solituder/OshiNote.git
cd OshiNote
pnpm install
```

### 開発実行

デスクトップアプリ全体を起動：

```bash
pnpm tauri dev
```

Vite フロントエンドのみを起動：

```bash
pnpm dev
```

一部ページは Tauri の SQLite とファイルシステム API に依存しています。実際の機能確認には `pnpm tauri dev` を推奨します。Vite 単体は主に UI の素早い確認向けです。

### 品質チェック

```bash
pnpm quality
```

このコマンドは次を順番に実行します。

```bash
pnpm lint
pnpm build
```

個別に実行することもできます。

```bash
pnpm lint
pnpm build
```

### ビルド

デスクトップアプリをビルド：

```bash
pnpm tauri build
```

インストーラーを生成せず、デスクトップ実行ファイルだけを検証：

```bash
pnpm tauri build --no-bundle
```

フロントエンドの静的ファイルのみをビルド：

```bash
pnpm build
```

デスクトップのビルド成果物は `src-tauri/target/` に生成されます。このディレクトリは Git にコミットしないでください。

## データとプライバシー

- OshiNote はデフォルトでデータをローカルに保存します。
- プロフィール、ノート、タグ、手帳、設定などの主要データはローカル SQLite データベースに保存されます。
- 取り込んだ画像資料は、たとえば `media/illustrations/originals/{year}` や `media/illustrations/thumbnails/{year}` のようなアプリデータディレクトリにコピーされます。
- 画像資料を削除すると、アプリがコピーしたローカルメディアファイルも削除されます。
- ローカルデータベース、個人のメディアファイル、一時ログ、ビルド成果物はコミットしないでください。

## 開発メモ

- push 前にはできるだけ `pnpm quality` を実行してください。
- データベース、メディア、ファイルシステム、デスクトップ権限に関わる機能は Tauri runtime 上で確認してください。
- ローカル計画、一時テストログ、個人メディア、ローカルデータベース、ビルド成果物は Git に含めないでください。
- ルート互換性は重要です。古い手帳ルートは新しいトップレベル手帳モジュールへ自然に遷移できる必要があります。
- UI 変更後は、テーマやアニメーション速度を変えながら、ドロップダウン、ドロワー、ダイアログ、詳細パネル、全画面プレビューを確認してください。

## 検索について

現在のノート検索は、タイトル、リッチテキスト内容、プレーンテキスト内容、タグに実際に保存された文字列をそのまま検索します。

まだ日本語の読み正規化や、かな/漢字の自動展開には対応していません。たとえばノート内に `お疲れ様` と書かれていても、現時点では `おつ` では検索にヒットしません。

今後、読み辞書やカスタム別名を追加することで、`おつ` と `お疲れ様` のようなよく使う表現をまとめて検索できるようにする可能性があります。

## Roadmap

- リソース管理：すべてのノート、画像資料、テンプレート、再利用可能な素材を一元管理。
- テンプレートシステム：Note や Journal がテンプレートを使用した時点のスナップショットを保存し、元テンプレートが変更・削除されても古い内容を安定させる。
- スタンプシステム：Note、Illustration、Postcard、Journal Page に保存時スタンプまたは自由配置スタンプを追加。
- 手帳強化：より豊富なページテンプレート、本/ポストカードの収納フロー、キャンバス編集体験の改善。
- エクスポートとバックアップ：JSON、Markdown、TXT、メディアエクスポート、復元フロー。
- パフォーマンス改善：ルート単位のコード分割、大規模リストの仮想化、メディア読み込み検査、メモリ/リソース監査。
