OshiNote 是一款本地优先的桌面应用，用来保存、整理和重新回看与“推”有关的记忆。

它不是通用笔记软件，也不是社交发布工具。它的核心是一个私人的记忆工作台：你可以建立推的空间，记录看完直播、作品或活动后的心情，收藏图像资料，制作手账页面，并逐渐把零散的感受整理成属于自己的回忆库。

> 产品目标：让推活记忆变得可检索、可整理、好看，并且值得长期保存。

## 这个软件旨在做什么

- 记录直播感想、活动回忆、作品感受和日常推活碎片。
- 将笔记、图像、标签、归档和手账页关联到具体的推。
- 制作带有手作感的 Journal 页面：目前有笔记、图像和印章，后续会加入胶带、纸条、贴纸和明信片模板。
- 默认本地保存数据，把隐私和长期拥有权放在核心位置。
- 将英语、简体中文、日语作为一等 UI 语言支持。

## 当前阶段

OshiNote 目前处于早期可用阶段。

桌面外壳、本地 SQLite 存储、推空间、笔记编辑器、图像库、手账模块、资源骨架、印章流程、主题系统和三语 UI 都已经建立。接下来的重点会从“基础管理”转向“更有手作感的 Journal 创作体验”。

## 已完成

### App Shell

- Tauri 桌面外壳，自绘顶部窗口栏。
- 我的推侧边栏支持调整宽度。
- 侧边栏底部有 Home、Export、Settings。
- 顶部入口包含 Journal 和 Resources。
- 支持多主题、玻璃效果、界面字号和动画速度设置。

### 三语支持

- 支持 English、简体中文、日本語。
- 语言选择会保存在本地。
- 新增功能原则上需要同步补齐三语文案。

### 推空间

- 创建和管理推的资料。
- 保存头像、简介、主题色和相关链接。
- 每个推下有概览、笔记、图像和标签。
- 推主页展示最近笔记、最近图像和基础统计。

### 笔记

- 基于 TipTap 的富文本编辑器。
- 支持加粗、斜体、下划线、删除线、字号、字体、文字颜色、高亮和表情。
- 支持日期、归档、所属推、来源链接、标签和附件。
- 实时字符数和字数统计。
- 输入标签时会根据历史标签显示候选，减少同义标签手误。
- 推的笔记页支持归档 tabs 和归档管理。
- 支持搜索、网格/列表浏览、收藏、保存、删除和图片附件。

### 图像

- 支持导入 PNG、JPEG、WebP。
- 导入文件会复制到本地 app data 目录。
- 支持全局图像页和单个推下的图像页。
- 支持全部 / 官方 / 同人图 / 收藏筛选。
- 支持卡片、网格和列表布局。
- 支持详情面板和 fullscreen 预览。
- 删除图像记录时会同步删除复制保存的本地媒体文件。

### Journal / 手账

- Journal 是顶层模块，不再绑定在某个推下面。
- 支持 books 和 loose pages。
- Journal 编辑器可以在画布上排列笔记和图像。
- Page Setup 抽屉支持标题、日期、描述、背景、所属推和收纳设置。
- Journal 页面支持保存和重开。
- 旧的推内手账路由会兼容跳转到新的顶层 Journal 模块。

### Resources / Templates

- 已有资源相关路由：
  - `/resources`
  - `/resources/templates`
  - `/resources/materials`
- 已有只读内置模板展示。
- 已有非破坏式 `templates` 表。
- Materials 目前仍是方向占位，还不是完整素材库。

### 印章系统

- Note、Illustration、Journal Page 支持每个目标一枚印章。
- 印章设置已从右侧栏改为独立弹窗。
- 支持圆章、日期章、蜡封章、纸质标签、篆书章、花边章、行书/书写字体相关样式等。
- 支持自由盖章流程：配置印章、点击盖章、移动 ghost、左键落章。
- 支持大小、角度、透明度、颜色和文字设置。
- 弹窗内有实时预览。
- 印章支持保存、清除和重开恢复。
- 可选印章字体可以在设置里自动下载到 app data 目录。
- PNG 印章导入已进入计划，但还没有实现。

## 下一阶段计划

### 1. Journal Handmade Elements

下一阶段最适合推进 Journal 的手作感。

推荐顺序：

1. 胶带 v1
2. 纸条 / 便签 v1
3. 贴纸 / 素材 v1

胶带适合最先做，因为它数据简单、视觉收益高，也能验证自由放置、旋转、缩放和保存重开这套交互。

### 2. 明信片模板基础

明信片抽卡不应该一开始就做成随机拼图，而应该建立在固定模板之后。

一个明信片模板应该定义 slot，例如：

- 图片
- Note 摘要或摘句
- 日期
- 标签
- 装饰元素
- 留白区域

有了这些版式规则之后，再从笔记、图像、标签、归档、收藏或时间范围里随机填充内容。

### 3. 明信片抽卡

抽卡流程后续应该是：

- 选择模板
- 选择卡池
- 随机填充模板
- 预览明信片
- 重新抽取
- 保存到 Journal 页面

### 4. LLM 卡池理解

LLM 更适合作为第二阶段增强，而不是 v1 的核心依赖。

适合交给 LLM 的任务：

- 将语义相近的 tags 聚合成主题卡池。
- 从长 Note 中提取适合明信片的一句话。
- 根据内容推荐模板。
- 生成短标题。
- 判断内容主题，例如纪念日、治愈、直播回顾、观影回忆、月度总结。

## 开发

安装依赖：

```bash
pnpm install
```

运行桌面应用：

```bash
pnpm tauri dev
```

仅运行前端预览：

```bash
pnpm dev
```

部分功能依赖 Tauri 的 SQLite、文件系统和窗口 API。真实功能测试请优先使用 `pnpm tauri dev`。

质量检查：

```bash
pnpm lint
pnpm build
```

或：

```bash
pnpm quality
```

构建桌面应用：

```bash
pnpm tauri build
```

## 数据与隐私

- 默认本地保存数据。
- 推资料、笔记、标签、手账、模板、印章和设置会保存在 SQLite 或本地偏好中。
- 导入图像会复制到本地 app data 目录。
- 可选印章字体会保存到 app data 目录。
- 本地数据库、个人媒体、日志和构建产物不应提交到 Git。
